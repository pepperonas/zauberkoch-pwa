## Projekt: Zauberkoch рҹ§‘вҖҚрҹҚірҹҚё вҖ” KI-Rezept- & Cocktail-Generator

Mobile-first Web-App unter **https://zauberkoch.de**: Nutzer wГӨhlt Modus (Kochen/Cocktail), LГӨnderkГјche, Geschmacksrichtungen und Constraints in einem 3-Schritt-Wizard вҖ” die Claude-API generiert ein vollstГӨndiges Rezept, das sich **live per SSE-Streaming aufbaut** (Titel вҶ’ Zutaten вҶ’ Schritte). Repo: `pepperonas/zauberkoch-pwa` (privat). **Offene Selbstbedienungs-Registrierung** via Google (`OPEN_SIGNUP=true`); Allowlist bleibt als optionales Gate, falls je wieder geschlossen (`OPEN_SIGNUP=false`). Invite-Codes wurden entfernt (2026-07-12).

## Tech-Stack

| Layer | Technologie |
|---|---|
| Backend | Python 3.12, FastAPI, SQLite (WAL), SQLAlchemy 2, Pydantic v2, Alembic |
| KI | Anthropic API, Modell via env `ANTHROPIC_MODEL` (Default `claude-sonnet-5`), Streaming вҶ’ semantische SSE-Events |
| Frontend | React 19, Vite, TypeScript strict, TanStack Query |
| Styling | Material 3 Expressive вҖ” **handgebaut** (Design-Tokens als CSS Custom Properties, KEIN MUI/Ant) |
| Animation | Motion (framer-motion-Nachfolger), echte Spring-Physik |
| Auth | Google OAuth 2.0 (Auth Code + PKCE, server-seitig), Sessions als httpOnly-Cookies, CSRF-Schutz |
| Deployment | VPS 69.62.121.168: systemd `zauberkoch-api` (Port **8742** loopback) + statischer Build in `/var/www/zauberkoch.de/`, vHost im zentralen nginx, certbot. **KEIN Docker.** |

Aufgewertete Signatur-Animationen: **Favorit-Reward** (`FavoriteButton.tsx`), **Hero-Moment** beim Fertigwerden (`GeneratePage`: Settle-Pop + Reveal-Sweep + Haptik + Stagger), **Conjure-Stage** 264px + Atmung + rotierende Wartetexte. **Route-/View-Transitions (2026-07-13, native View Transitions API, `src/state/viewTransition.tsx`):** `ViewTransitionProvider` treibt `document.startViewTransition` **manuell** (react-routers `viewTransition`-Option ist im deklarativen `<BrowserRouter>` ein No-Op вҖ” brГӨuchte den Data-Router). Feste Namen `zk-shared-motif`/`zk-shared-title` (`viewTransition.tsx`), gesetzt **nur** am aktiven Element (`activeId === recipe.id`) вҶ’ pro Snapshot eindeutig, lГ¶st automatisch den вҖһviele Items"-Perf-Punkt (nur die geklickte Karte wird benannt). CSS in `base.css`: Theme-Reveal-Regeln auf `.zk-theme-vt` gescopet (sonst wГјrgen sie den Route-Root-Crossfade ab); `::view-transition-group(zk-shared-*)` morpht mit MD3-Emphasized `cubic-bezier(.2,0,0,1)` 360 ms; Root-Content crossfadet (`zk-vt-fade-in/out`, 320/180 ms). **Kritische Mechanik (mГјhsam erarbeitet):** Der VT-Callback gibt fГјr **beide** Richtungen eine Promise zurГјck, die erst beim tatsГӨchlichen React-Route-Commit resolved вҖ” Гјber einen `pendingResolve`-Ref, den der Scroll-`useLayoutEffect` (auf `location.key`) auslГ¶st (+ 500 ms Safety-Timeout). **KEIN `flushSync(navigate)`**: react-router v7 committet die Navigation nicht synchron (deferred), und rAF ist wГӨhrend der VT-Callback-Phase ausgehungert вҶ’ beide Wege scheiterten (New-Snapshot = alte Seite вҶ’ spГӨter Swap вҶ’ Morph gecancelt). Drei weitere Fallen, die den Morph killten: (a) **Suspense** вҖ” die Ziel-Seite MUSS beim Commit synchron mit Hero da sein; eine lazy `RecipeDetailPage` suspendet einen Frame вҶ’ Fallback-Snapshot ohne Hero вҶ’ Cancel. LГ¶sung: **`RecipeDetailPage` wird per verstecktem Off-Route-Mount vorgewГӨrmt** (`{me && !'/rezept/' && <div hidden><Suspense><RecipeDetailPage/></Suspense></div>}` in App.tsx) вҶ’ `React.lazy` ist vor der Navigation resolved, Route rendert synchron вҖ” OHNE die Seite (RecipeView/RecipeMotif, ~12 KB gz) in den Entry-Chunk zu ziehen (Lighthouse!). (b) **Query** вҖ” `RecipeCard` `ensureQueryData` VOR der Navigation (sonst Ladezustand ohne Hero), `onPointerDown`-Prefetch macht das Warten ~0. (c) **framer-Hero-Entrance** вҖ” der `riseIn` der `motion.section.hero` wird bei `isShared` unterdrГјckt (animiert den Motiv-Container und wГјrgte die VT ab). **Perf:** kein `backdrop-filter`-Jank mehr (VT snapshottet den Header einmalig вҖ” der `.zk-page-vt`-Blur-Drop entfiel). **Scroll:** `history.scrollRestoration='manual'`; Position wird in `go()` synchron VOR der Navigation gemerkt (ein Scroll-Listener wГјrde mit dem VorwГӨrts-`scrollTo(0,0)` racen und 0 speichern), Restore/Reset im `useLayoutEffect` auf `location.key` (POP вҶ’ gemerkt, sonst 0). **Fallback:** `prefers-reduced-motion` oder fehlendes `startViewTransition` вҶ’ normale Navigation ohne VT.
